Schedule 10.4 Revisit – Overview

Visits 2, 3, and 4 (Months 2-4 of Panel)

Primary Focus

  • Membership Changes: Track births, deaths, migrations
  • Current Weekly Status: Update for new reference week
  • Efficiency: Shorter questionnaire
  • Panel Stability: 75% overlap month-to-month

What's Different

  • Shaded Blocks: Many blocks not filled
  • No Repeat: Usual status not re-collected
  • No Vocational: Block 4.1 skipped
  • Limited HH Char: Only size updated
Purpose: Revisits enable production of monthly CWS indicators by tracking the same households over time while reducing respondent burden and survey costs.
Visit Month Key Task Blocks Completed

Revisit 1 2 Changes + CWS 0, 1, 2, 3(partial), 4, 6
Revisit 2 3 Changes + CWS 0, 1, 2, 3(partial), 4, 6
Revisit 3 4 Changes + CWS 0, 1, 2, 3(partial), 4, 6

Schedule 10.4 Revisit – Block 4

Demographics – Membership Changes

Part A: Erstwhile Members

  • Copy from First Visit: Serial number, Name, Age
  • Update Status: Membership status code
    • 1: Still member (present)
    • 2: Died
    • 3: Moved out permanently
    • 4: Temporarily away
    • 5: Other (specify)
  • Age Update: Update if needed (birthday occurred)

Part B: New Members

  • Births: Since First Visit
  • In-migration: New residents
  • Marriage: New spouse joined
  • Full Details: Same as First Visit Block 4
    • Name, Relation, Gender
    • Age, Marital Status
    • Education, Attendance
Block 4.1 (Vocational): Completely shaded in Revisits. Not filled even for new members. Vocational data collected only in First Visit.
Tracking Tip: If entire household moved, determine if within FSU. If yes, continue. If moved out of FSU, treat as casualty.

Schedule 10.4 Revisit – Blocks 5.1–5.3

All Usual Status Blocks are Shaded/Not Filled in Revisits
Block Content Revisit Status Reason
5.1 Usual Principal Activity Shaded 365-day reference unchanged over 4 months
5.2 Usual Subsidiary Activity Shaded Part of usual status
5.3 Follow-up Questions Shaded Relates to usual status
Guideline: Usual activity status is collected only in First Visit. It represents activity over 365 days, which doesn't change significantly over the 4-month panel period. Re-collecting would be redundant.

What This Means

  • No employment status updates
  • No industry/occupation changes
  • No usual time spent updates
  • No job search updates

What IS Collected

  • Block 6: Current Weekly Activity
  • Captures short-term changes
  • Enables monthly unemployment rates
  • Tracks labour market transitions

Schedule 10.4 Revisit – Key Guidelines

Critical Instructions for Field Staff

1
Track Membership Changes Carefully
Document births, deaths, migrations with dates and reasons
2
Age Updates
Copy from First Visit but update if birthday occurred between visits
3
No Repeat Collection
Do NOT re-collect usual status, vocational training, or detailed household characteristics
4
Household Split Protocol
If household splits, canvass senior-most split within FSU; treat as casualty if none remain
5
Current Weekly Status Priority
Block 6 is the primary output of revisits – ensure accurate day-by-day recording
Casualty Rules: If household cannot be located after 3 attempts, or refuses participation, or has moved out of FSU, record as casualty with appropriate reason code.
Output: Revisit data enables production of monthly and quarterly CWS indicators, tracking labour market dynamics over time.
